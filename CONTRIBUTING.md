# Contributing to THE FORGE

Welcome! THE FORGE is a community-driven methodology kit for turning AI coding assistants into disciplined Research Engineers. We welcome contributions that add new language stacks, improve existing EVAL harnesses, or refine the core methodology.

## Ways to Contribute

1.  **Add a New Stack:** Create a new folder under `templates/stacks/` with an `EVAL.sh` and `EVAL_SPEC.md` for a language or framework not yet supported.
2.  **Improve EVAL Harnesses:** Enhance existing `EVAL.sh` scripts with better default quality checks or more robust error handling.
3.  **Refine Methodology:** Propose changes to the universal operating rules in `templates/universal/FORGE.md`.
4.  **Bug Fixes:** Fix issues in the adoption or update scripts.
5.  **Documentation:** Improve the clarity and completeness of the docs.

## Development Workflow

1.  **Fork and Clone:** Create your own fork of the repository.
2.  **Branch:** Create a feature branch for your changes.
3.  **Test:** If you're adding a stack, ensure the `EVAL.sh` runs correctly in a test repository using the adoption scripts.
4.  **Submit PR:** Open a Pull Request with a clear description of your changes.

## Code of Conduct

Please be respectful and professional in all interactions. We aim to build a supportive community of engineers pushing the boundaries of AI-assisted development.
